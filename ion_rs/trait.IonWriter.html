Expand description

This trait captures the format-agnostic encoding functionality needed to write native Rust types to a stream as Ion values.

fn supports_text_symbol_tokens(&self) -> bool

Returns true if this RawWriter supports writing field names, annotations, and symbol values directly as text; otherwise, returns false.

If this method returns false, passing a crate::RawSymbolToken::Text to the Self::set_annotations, Self::set_field_name, or Self::write_symbol methods may result in a panic.

Sets the current field name to name. If the TextWriter is currently positioned inside of a struct, the field name will be written before the next value. Otherwise, it will be ignored.

fn flush(&mut self) -> IonResult<()>

Causes any buffered data to be written to the underlying io::Write implementation. This method can only be called when the writer is at the top level.

fn output(&self) -> &Self::Output

Returns a reference to the writer’s output.

This method can be used to inspect the Ion data that the writer has produced without having to first drop the writer.

use ion_rs::element::Element;
use ion_rs::{IonResult, IonWriter, TextWriter, TextWriterBuilder};
// Set up our output buffer
let mut buffer: Vec<u8> = Vec::new();
// Construct a writer that will serialize values to the buffer
let mut writer = TextWriterBuilder::default().build(&mut buffer)?;
// Serialize some data
writer.write_string("hello")?;
writer.flush()?;
// Read the data back from the output buffer
let output_element = Element::read_one(writer.output())?;
// Confirm that it matches the input data
assert_eq!(Element::from("hello"), output_element);
source

fn output_mut(&mut self) -> &mut Self::Output

Returns a mutable reference to the writer’s output.

Modifying the underlying sink is an inherently risky operation and can result in unexpected behavior or invalid data. It is not recommended for most use cases.
